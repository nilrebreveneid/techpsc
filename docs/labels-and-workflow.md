## Working model

Use issues for planning and recordkeeping.

Use pull requests for:

- code submissions
- document submissions
- content updates
- file-based deliverables

Use update issues for:

- weekly reporting
- async status reporting
- non-code deliverables

Use finance issues for:

- reimbursements
- expense approvals
- payment tracking
- budget logging

## Suggested review policy

- At least one reviewer for every pull request
- Points awarded only after reviewer approval
- Finance requests reviewed by finance lead or admin
- Use CODEOWNERS after GitHub usernames and teams are finalized
