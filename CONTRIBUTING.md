# Contributing To TECHPSC

This repository uses GitHub Issues, Pull Requests, and Projects for all work.

## Workflow

1. Create or pick an issue.
2. Make sure the issue has an assignee, category, priority, and points value if contributor work is involved.
3. Move the item to `In Progress` in the GitHub Project.
4. Create a branch for the work.
5. Open a pull request using the PR template.
6. Add proof of work, links, screenshots, or files as needed.
7. A reviewer checks the submission.
8. After approval, merge the PR and mark the project item `Done`.
9. Update payment or reward status if applicable.

## Contributor points

Track contributor points in the GitHub Project using the `Points` custom field.

Use points for:

- completed assigned work
- reviewed and accepted submissions
- documentation or operational contributions

Do not assign points until the submission is reviewed.

## Finance-related work

Use the finance issue form for:

- reimbursements
- budget requests
- expense approvals
- invoice tracking
- payment follow-up

Finance records should be tracked with:

- `Budget`
- `Payment Status`
- `Proof Link`
- `Approved By`

## Communication rules

- Use issues for tasks and records.
- Use pull requests for submissions and code reviews.
- Use discussions or weekly update issues for announcements and progress updates.
- Keep all links, screenshots, and proofs inside GitHub so work is auditable.
